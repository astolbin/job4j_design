Придумайте 3 примера, когда происходит нарушение принципа ISP:
1. если описать в CRUD-интерфейсе все возможные операции (create, update, get, delete), а потом попытаться реализовать интерфейс в модели, которая, например, не будет удаляться - метод delete придется оставить пустым
2. интерфейс Message, в котором есть зависимость на несколько сервисов (и отправка через email, и sms, и push)
3. интерфейс для вывода информации об обьектах View, требующий реализацию нескольких форматов (html, xml и т.д.)
